# 软件需求规格说明书 (SRS)

> **项目名称：** AstroLog — 天文观测日志与星体管理系统
> **版本：** v1.0
> **编制日期：** 2026-05-13
> **技术栈：** Java SE 17 + Swing + MySQL 8.0 + JFreeChart

---

## 1. 引言

### 1.1 编写目的

本文档旨在详细定义 AstroLog 天文观测日志与星体管理系统的功能需求和非功能需求，作为后续架构设计、编码实现和测试验收的基准依据。

**预期读者：** 开发人员、测试人员、项目评审教师。

### 1.2 项目背景

天文爱好者或社团在长期观星活动中，需要系统化管理观测记录、星体信息及器材设备。传统纸质记录或 Excel 表格难以进行关联查询与统计分析，因此开发一款桌面应用，实现观测日志的数字化管理。

### 1.3 产品范围

AstroLog 是一款基于 Java Swing 的桌面应用程序，面向天文爱好者群体，提供星体信息浏览、观测记录管理、器材管理、数据可视化和报告导出等功能。系统支持观测者和管理员两种角色，采用本地 MySQL 数据库存储。

### 1.4 术语定义

| 术语              | 说明                                       |
| ----------------- | ------------------------------------------ |
| 观测者 (Observer) | 普通用户角色，可管理个人观测记录与器材     |
| 管理员 (Admin)    | 拥有全部权限，可维护星体库和用户账号       |
| 视宁度 (Seeing)   | 大气稳定度指标，1-5 级，1 级最佳           |
| Bortle 等级       | 光害等级，1-9 级，1 级最暗（最佳观测条件） |
| 赤经 (RA)         | Right Ascension，天球经度，用时/分表示     |
| 赤纬 (Dec)        | Declination，天球纬度，用度/分表示         |
| 梅西耶天体        | 法国天文学家梅西耶编录的 110 个深空天体    |

## 2. 总体描述

### 2.1 产品视角

AstroLog 是一个独立运行的桌面应用程序，不依赖网络服务。系统通过 JDBC 直连本地或局域网 MySQL 数据库。未来可扩展为 C/S 架构或迁移至 Web 平台。

### 2.2 用户特征

| 角色   | 特征                               | 关键需求                           |
| ------ | ---------------------------------- | ---------------------------------- |
| 观测者 | 天文爱好者，具备基本计算机操作能力 | 快速记录观测、浏览星体库、查看统计 |
| 管理员 | 社团骨干或教师，负责数据维护       | 维护星体数据、管理用户、系统配置   |

### 2.3 约束条件

- 运行环境：Windows 10+ / macOS / Linux，需安装 JDK 17+ 和 MySQL 8.0
- 界面框架：Java Swing（不引入 JavaFX 或 Web 框架）
- 数据存储：MySQL 关系型数据库，字符集 utf8mb4
- 开发模式：AI 辅助编码 + 人工逻辑校验与代码审查
- 体量要求：不低于 12,000 行 Java 源代码（租车系统 4,200 行的 3 倍）

### 2.4 假设与依赖

- 用户已安装 MySQL 8.0 并创建 astrolog 数据库
- Maven 可用，能够下载 JFreeChart、JasperReports、jBCrypt 等第三方依赖
- 静态天文数据（梅西耶目录、星座信息）以 JSON 资源文件内置

## 3. 功能需求

### 3.1 用户管理模块 (F1)

**F1.1 用户注册**

| 项目     | 描述                                                                                                                                               |
| -------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| 优先级   | 高                                                                                                                                                 |
| 参与者   | 所有用户                                                                                                                                           |
| 前置条件 | 未登录状态                                                                                                                                         |
| 基本流   | 1. 用户填写用户名、密码、确认密码、角色、城市、默认坐标 2. 系统校验用户名唯一性、密码强度 3. BCrypt 加密密码后写入数据库 4. 显示注册成功并跳转登录 |
| 异常流   | 用户名已存在 → 提示"用户名已被注册"；密码强度不足 → 提示密码要求；两次密码不一致 → 提示重新输入                                                    |

**F1.2 用户登录**

| 项目     | 描述                                                                                |
| -------- | ----------------------------------------------------------------------------------- |
| 优先级   | 高                                                                                  |
| 前置条件 | 已注册账号                                                                          |
| 基本流   | 1. 输入用户名密码 2. 系统验证 BCrypt 哈希 3. 检查账号是否锁定 4. 登录成功进入主界面 |
| 异常流   | 连续 5 次失败 → 账号锁定 30 分钟；账号已锁定 → 提示剩余解锁时间                     |

**F1.3 个人信息维护**

修改城市、默认观测坐标、头像。密码修改需验证原密码。

**F1.4 用户账号管理（管理员）**

查看所有用户列表、启用/禁用账号、重置密码。

### 3.2 星体信息管理模块 (F2)

**F2.1 星体信息 CRUD（管理员）**

增加/修改/删除星体信息，包括名称、类型（恒星/行星/星云/星团/星系）、所属星座、赤经赤纬坐标、视星等、最佳观测季节、文字描述、图片路径、梅西耶编号、NGC 编号、距离光年。

**F2.2 星体浏览与查询**

所有用户可按星座、类型、亮度范围、最佳观测季节组合筛选。支持关键词模糊搜索。

**F2.3 批量导入（管理员）**

支持 CSV 文件批量导入星体数据，含格式校验和重复检测。

**F2.4 收藏星体**

用户可收藏感兴趣的星体，在"我的收藏"中快速查看。支持按热度排序（被收藏次数）。

### 3.3 观测记录管理模块 (F3)

**F3.1 添加观测记录**

选择星体（必填）、观测时间（必填）、观测地点（可选）、天气、视宁度（1-5）、所用器材（多选）、标签（多选）、文字笔记、月相。

**F3.2 查询观测记录**

组合条件查询：时间范围、星体名称、星座、观测地点、天气、标签。结果以 JTable 展示，支持排序。

**F3.3 编辑与删除**

用户只能修改/删除自己的观测记录。管理员可维护全部记录。

**F3.4 观测计划**

创建计划观测清单，设定目标星体和预计观测日期，可从计划一键转为实际观测记录。

### 3.4 器材管理模块 (F4)

**F4.1 器材 CRUD**

型号名称、类型（望远镜/目镜/相机/其他）、口径、焦距、描述、购买日期、状态（在用/维修/退役）。

**F4.2 维护日志**

记录器材维护/维修历史：日期、描述、费用、下次维护日期。支持按器材筛选。

**F4.3 使用统计**

统计每件器材被关联的观测次数，展示使用频率排名。

### 3.5 数据可视化模块 (F5)

**F5.1 年度/月度观测频次柱状图**

按年或按月统计观测次数，JFreeChart 柱状图渲染。

**F5.2 星体类型分布饼图**

按恒星/行星/星云/星团/星系分类统计观测占比。

**F5.3 年度观测趋势折线图**

跨年度观测频次变化趋势（需至少 2 年数据）。

**F5.4 观测日历热力图**

类似 GitHub 贡献图风格，按日展示观测密度，颜色深浅代表观测次数。

**F5.5 观测条件雷达图**

天气、视宁度、月相、光害等多维度综合展示观测质量。

**F5.6 星座亮星分布图**

自定义 Canvas 绘制，以北极星为中心，标注主要星座亮星位置与连线。

### 3.6 报告导出模块 (F6)

**F6.1 年度观测报告**

模板包含：年度观测次数、最常观测星体 Top 10、观测类型分布、月度趋势、器材使用统计。导出格式：HTML（简单模板）和 PDF（JasperReports 模板）。

**F6.2 梅西耶证书**

完成全部 110 个梅西耶天体观测的用户可生成纪念证书（PDF）。

### 3.7 数据校验模块 (F7)

**F7.1 输入校验框架**

- 用户名：4-20 字符，字母/数字/下划线
- 密码：最少 8 字符，含大小写字母+数字+特殊字符
- 赤经/赤纬坐标：格式和范围校验
- 经纬度：合法范围校验 (-90~90, -180~180)
- 视宁度：1-5 整数

**F7.2 业务校验**

- 用户名唯一性
- 观测记录的时间不能在未来
- 器材删除前检查是否有关联观测记录

### 3.8 观测地管理模块 (F8)

观测地点 CRUD：名称、纬度、经度、海拔、Bortle 光害等级（1-9）、最佳观测时段描述。用户可在添加观测时快速选择已保存的地点。

### 3.9 星座百科模块 (F9)

内置 88 星座基础数据：名称、缩写、面积、亮星数、最佳观测季节。亮星列表展示（名称、星等、坐标）。深空天体目录（梅西耶、NGC 编号索引）。

### 3.10 系统配置模块 (F10)

- **主题切换：** 亮色模式/暗色模式/星空主题（三选一），所有面板同步切换
- **字体设置：** 界面字体大小调整（小/中/大）
- **数据备份与恢复：** 导出全部数据为 SQL 文件，从 SQL 文件恢复

### 3.11 今夜星空模块 (F11)

- 输入日期和观测位置（经纬度），自动推荐当晚可见的主要星座和亮星
- 展示当晚月相、月升/月落时间，计算黄金观测窗口
- 推荐结果可直接"开始观测"，跳转到添加观测记录界面

### 3.12 梅西耶马拉松追踪模块 (F12)

- 内置 110 个梅西耶天体完整目录（编号、名称、类型、星座、星等、季节）
- 按类型筛选（星云/星团/星系）、按观测状态筛选（已观测/未观测）
- 进度追踪：展示完成百分比和进度条
- 完成全部 110 个后自动解锁梅西耶证书

### 3.13 星座文化馆模块 (F13)

- 88 星座希腊神话故事浏览
- 中国星官体系对照（三垣二十八宿）
- 北斗七星等在不同文化中的名称和传说
- 星图交互展示（星座连线绘制）

## 4. 非功能需求

### 4.1 性能需求

| 指标         | 要求                   |
| ------------ | ---------------------- |
| 启动时间     | < 5 秒                 |
| 列表查询响应 | < 1 秒（单表万级数据） |
| 图表渲染     | < 2 秒                 |
| 内存占用     | < 512 MB               |

### 4.2 安全需求

- 密码 BCrypt 加盐哈希存储，不可逆
- 连续 5 次登录失败锁定 30 分钟
- SQL 注入防护：全部使用 PreparedStatement 参数化查询
- 操作日志：记录用户关键操作（登录/删除/修改权限）

### 4.3 可用性需求

- 界面遵循 Swing 平台 LookAndFeel 规范
- 关键操作（删除）弹出确认对话框
- 表单校验实时反馈（输入框边框变红，显示错误提示）
- 支持键盘快捷键（Ctrl+S 保存，Ctrl+F 搜索）

### 4.4 可维护性需求

- 三层架构严格分层，层间仅通过接口交互
- DAO 层通过 BaseDao 模板方法统一 JDBC 操作
- 包结构清晰，命名遵循 Java 社区规范
- 关键类和方法有 Javadoc 注释

### 4.5 国际化需求

- 界面文本通过 ResourceBundle 管理
- 首期提供中文（简体）和英文两套语言文件
- 后续可扩展其他语言
