# 开发执行计划

> **项目名称：** AstroLog — 天文观测日志与星体管理系统
> **版本：** v1.0
> **编制日期：** 2026-05-13
> **开发模型：** 混合模型 — 基础先行 + 垂直功能切片

---

## 1. 计划概述

### 1.1 开发模型

采用**混合模型**：阶段一完成基础设施搭建，阶段二以垂直功能切片迭代交付，阶段三完成扩展功能与收尾。

每个阶段末尾设 4 个人工校验关口（G1 代码审查 / G2 功能验收 / G3 测试覆盖 / G4 文档同步），通过后方可进入下一阶段。

### 1.2 总体进度

| 阶段     | 名称           | 子阶段数 | 预估文件 | 预估代码行  |
| -------- | -------------- | -------- | -------- | ----------- |
| 一       | 基础奠基       | 2        | ~33      | ~3,700      |
| 二       | 功能切片迭代   | 4        | ~36      | ~6,200      |
| 三       | 扩展功能与收尾 | 3        | ~31      | ~4,200      |
| **合计** |                | **9**    | **~100** | **~14,100** |

## 6. 风险管理

| 风险                              | 概率 | 影响 | 应对措施                                   |
| --------------------------------- | ---- | ---- | ------------------------------------------ |
| AI 生成代码质量不稳定             | 中   | 高   | 每阶段 G1 关口严格审查，必要时重写         |
| JFreeChart/JasperReports 集成问题 | 低   | 中   | S6/S9 预留缓冲时间，提前验证 API 兼容性    |

| 体量不达标                        | 低   | 高   | 在 S1-S6 阶段持续统计代码行数，及时补充    |
