## 2. 单元测试

### 2.1 测试框架

- **JUnit 5** (5.10+) — 测试运行框架
- **Mockito** (5.7+) — Mock 依赖隔离
- **JaCoCo** — 代码覆盖率报告

### 2.2 覆盖目标

## 5. 人工校验关口

### 5.1 关口定义

每个开发阶段结束后，在进入下一阶段前必须通过 4 个校验关口。

### 5.2 G1 — 代码审查

| 检查项   | 标准                                                  |
| -------- | ----------------------------------------------------- |
| 包结构   | 符合架构设计文档定义                                  |
| 命名规范 | 类名 PascalCase、方法名 camelCase、常量 UPPER_SNAKE   |
| 三层隔离 | UI 不直接调用 DAO，Service 不处理 Swing 组件          |
| 设计模式 | 策略/观察者/工厂/单例/模板方法正确实现                |
| 安全漏洞 | 无 SQL 注入（全 PreparedStatement）、密码 BCrypt 加密 |
| 资源管理 | JDBC 连接在 finally 中关闭，无泄漏风险                |
| 重复代码 | 无明显复制粘贴代码，公共逻辑已提取                    |

**工具：** AI 代码审查 agent + 人工逐项确认

### 5.3 G2 — 功能验收

对照 SRS 功能需求逐项验收。使用手动测试清单逐项打勾。验收标准：所有 P0 功能通过，P1 功能 95% 通过。

### 5.4 G3 — 测试覆盖

- `mvn test` 全部通过，零失败
- `mvn jacoco:report` 覆盖率报告
- Service 层行覆盖率 ≥ 85%
- Util 层行覆盖率 ≥ 80%
- 整体行覆盖率 ≥ 70%
- 所有集成测试通过

### 5.5 G4 — 文档同步

- 对比代码实现与设计文档，确认无偏差
- 如有偏差：评估是否需要更新文档或修正代码
- 检查 API 文档（Javadoc）是否覆盖所有 public 方法
- 检查 SQL 脚本是否与数据库实际表结构一致
