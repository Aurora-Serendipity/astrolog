## 3. 与交接文档的偏差说明

| 偏差项                                 | 说明                                                                                                                         |
| -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **MainFrame 提前修改**                 | 原计划第 9 步修改，因 LoginFrame 编译依赖提前至第 6 步后，仅做构造器+字段最小改动                                            |
| **DashboardPanel 保留 MainFrame 参数** | 交接文档写 `DashboardPanel(User user)`，实际改为 `DashboardPanel(User user, MainFrame mainFrame)` 以保留导航能力             |
| **UserService 新增测试构造器**         | 新增 `UserService(UserDao, LogDao)` package-private 构造器，使 Mockito @InjectMocks 能注入                                   |
| **pom.xml 新增 surefire**              | 添加 `-Dnet.bytebuddy.experimental=true` 解决 Mockito/Byte Buddy 不支持 Java 25 的问题                                       |
| **UserDaoTest 调整**                   | BigDecimal 比较改用 `compareTo()`（避免 MySQL DECIMAL 精度差异），大小写测试改用不存在的用户名（MySQL utf8mb4 大小写不敏感） |
| **LogDaoTest 调整**                    | 通过 UserDao 创建真实测试用户以满足 operation_logs 外键约束                                                                  |

## 6. S3 准备

S2 的 LoginFrame 登录成功后已传递完整 `User` 对象给 `MainFrame`，MainFrame 通过 `getCurrentUserObject()` 暴露当前用户。S3（星体管理）只需从 MainFrame 获取当前用户即可，无阻塞依赖。
