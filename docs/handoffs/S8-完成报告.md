## 4. 与交接文档的偏差

| 偏差项                              | 说明                                                                                                                  |
| -------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| JSON 文件行数远超估算                    | 交接估算 \~300 + \~250 = \~550 行，实际 992 + 706 = 1,698 行。原因：每个条目含 7-9 行 JSON（多字段 + description/mythology 长文本），是数据密度非行数问题 |
| Service 类行数偏低                    | MessierService 83 行 vs \~160 估算（-48%），ConstellationService 47 行 vs \~100 估算（-53%）。交接估算偏保守                           |

## 5. 核心架构决策

- **MessierService 数据源：** 110 条梅西耶目录从 messier\_catalog.json 一次性加载到内存，无数据库表（观测状态从 observations JOIN celestial\_bodies.messier\_number 推导）
- **MessierPanel 状态列渲染：** 分离 StatusRenderer（第 7 列 ✓/○ 颜色+字体）和 RowColorRenderer（第 1-6 列行背景色），避免列索引偏移
- **ConstellationPanel 双视图：** CardLayout 切换 JTable 列表和 GridLayout 网格，共享同一数据模型，搜索结果同步刷新
- **ConstellationPanel 星图交互：** "\[在星图中查看]" 按钮用于在新 JFrame 中通过 BodyDao.findAll() 获取天体并嵌入 StarMapCanvas
