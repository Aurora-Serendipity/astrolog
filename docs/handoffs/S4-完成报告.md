## 4. S4 开发偏差记录

| 偏差                                       | 原因                                        | 影响                                                                 |
| ------------------------------------------ | ------------------------------------------- | -------------------------------------------------------------------- |
| G1 发现 N+1 查询                           | EquipmentPanel 每行单独调用 getUsageCount() | 新增 EquipDao.batchGetUsageCounts() 批量方法，同时解决了三段重复代码 |
| G1 发现 showEquipDialog 缺少异常处理       | 口径/焦距/购买日期字段未包裹 try-catch      | 新增 NumberFormatException + DateTimeParseException 统一捕获         |
| G1 发现 hasSameType 死代码                 | 未调用且语义歧义                            | 删除方法，替换为 batchGetUsageCounts                                 |
| G1 发现 updateEquipment 缺少 type 验证     | 与 addEquipment 不一致                      | 新增 type 空值校验                                                   |
| pom.xml surefire argLine 需 @{argLine}     | 硬编码 argLine 覆盖 JaCoCo agent            | 新增 @{argLine} 占位符                                               |
| java.util._ 与 java.sql._/java.awt.\* 冲突 | 通配符导入导致 Date/List 歧义               | 改用显式 import                                                      |
| G2-11 权限测试场景不可达成                 | 器材列表仅显示当前用户数据（设计正确）      | UI 层所有权检查识别为纵深防御，G1 审查通过即可                       |
| DashboardPanel 字体方框                    | Segoe UI Emoji 不含 Unicode 符号（★ 等）    | 改用 Font.SANS_SERIF（与 S3 偏差一致）                               |
| G2-7 排序验证需观测数据                    | obs_equipment 有 FK 到 observations 表      | S5 后可端到端验证，当前通过 SQL 手动造数据                           |
